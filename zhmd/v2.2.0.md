# Nunchaku Z-Image-Turbo LoRA 加载器 V3 - 完整技术文档

## 开发背景

我们之前为自定义的非官方 Nunchaku Z-Image Turbo 加载器开发了一个 LoRA 加载器原型（[非官方加载器测试记录](https://github.com/ussoewwin/ComfyUI-QwenImageLoraLoader/blob/main/md/technical_explanation.md)）。随着 ComfyUI-nunchaku 官方 Z-Image-Turbo 模型加载器（采用 NextDiT）的发布，我们重构了该 LoRA 加载器以使其与官方模型加载器协同工作。这并不是因为官方模型加载器存在任何问题，而是为了使我们现有的原型适应 ComfyUI-nunchaku 的官方实现。

**对于 ComfyUI-nunchaku 的官方 Z-Image-Turbo 模型加载器，请使用 V3 版本。V2 版本仅供使用我们自定义的非官方加载器进行测试。**

为了进行适配，我们解决了以下差异：

**模型结构差异**

- 自定义非官方加载器：`NunchakuZImageTransformer2DModel`（diffusers 格式，`to_qkv` / `feed_forward.net.0.proj`）
- ComfyUI-nunchaku 官方模型加载器：`NextDiT`（ComfyUI Lumina2 格式，`attention.qkv` / `feed_forward.w1/w2/w3` 或 `w13`）

**前向传播签名差异**

- 自定义非官方：`forward(x_list: List[torch.Tensor], t_zimage, cap_feats=...)`
- ComfyUI-nunchaku 官方：`forward(x: torch.Tensor, timesteps, context, num_tokens, attention_mask=...)`

**LoRA 映射不匹配**

- 现有的 V2 加载器是针对自定义非官方结构（`to_qkv` / `net.0.proj`）设计的，无法解析 ComfyUI-nunchaku 官方结构（`qkv` / `w13`）的键。

**保留 V2 以供测试**

- 保留 V2 代码，用于我们自定义 of 非官方加载器的测试。
- 提供 V3 作为一个新文件，以与 ComfyUI-nunchaku 的官方模型加载器协同工作。
- 两者均使用相同的 `compose_loras_v2`，但通过自动映射切换来正确运行。

**修改 1：NextDiT 映射定义（第 30-54 行）**

```python
ZIMAGE_NEXTDIT_UNPATCHED_KEY_MAPPING = [
    (re.compile(r"^(layers)[._](\d+)[._]attention[._]to[._]([qkv])$"), r"\1.\2.attention.qkv", "qkv", lambda m: m.group(3).upper()),
    (re.compile(r"^(layers)[._](\d+)[._]attention[._]to[._]out(?:[._]0)?$"), r"\1.\2.attention.out", "regular", None),
    (re.compile(r"^(layers)[._](\d+)[._]feed_forward[._]w1$"), r"\1.\2.feed_forward.w1", "regular", None),
    (re.compile(r"^(layers)[._](\d+)[._]feed_forward[._]w2$"), r"\1.\2.feed_forward.w2", "regular", None),
    (re.compile(r"^(layers)[._](\d+)[._]feed_forward[._]w3$"), r"\1.\2.feed_forward.w3", "regular", None),

ZIMAGE_NEXTDIT_NUNCHAKU_PATCHED_KEY_MAPPING = [
    (re.compile(r"^(layers)[._](\d+)[._]attention[._]to[._]([qkv])$"), r"\1.\2.attention.qkv", "qkv", lambda m: m.group(3).upper()),
    (re.compile(r"^(layers)[._](\d+)[._]attention[._]to[._]out(?:[._]0)?$"), r"\1.\2.attention.out", "regular", None),
```

**技术含义**：

- 指定为 `glu` 组后，会通过 `_fuse_glu_lora` 合并并应用 w1 和 w3。

## 技术设计哲学

### 1. 自动映射切换

**问题**：自定义非官方加载器（`to_qkv`/`net.0.proj`）与 ComfyUI-nunchaku 官方加载器（`qkv`/`w13`）之间的模块名称不同。

**解决方案**：

- 使用全局变量 `_ACTIVE_KEY_MAPPING` 在运行时切换映射。
- 在 `compose_loras_v2` 内部检测模型结构并设置适当的映射。
- 在 `_classify_and_map_key` 中优先采用 `_ACTIVE_KEY_MAPPING`。

**优势**：

- V2/V3 共享同一个 `compose_loras_v2` 函数。
- 最小化代码重复。
- 要支持新的模型结构，只需添加相应的映射定义。

### 2. 动态前向传播签名检测

**问题**：自定义 Z-Image（`cap_feats`）与 ComfyUI-nunchaku 官方 NextDiT（`context`/`num_tokens`）的前向传播签名不同。

**解决方案**：

- 使用 `inspect.signature` 动态获取前向传播参数。
- 若存在 `cap_feats` 则跳转到 Z-Image 处理分支，若存在 `context` 和 `num_tokens` 则跳转到 NextDiT 分支。

**优势**：

- 单个 `ComfyZImageTurboWrapper` 即可处理两种格式。
- 无需进行类型检查或 isinstance 检查（具有极高的灵活性）。

### 3. GLU 融合处理

**问题**：ComfyUI-nunchaku 将 `feed_forward.w1/w3` 合并为 `w13`。

**解决方案**：

- `ZIMAGE_NEXTDIT_NUNCHAKU_PATCHED_KEY_MAPPING` 将 LoRA 的 `w1`/`w3` 映射到 `w13`。
- 将其组指定为 `glu`，并通过 `_fuse_glu_lora` 以融合方式应用。

**技术细节**：

- `_fuse_glu_lora` 函数用于在数学上合并 w1 and w3 LoRA 并将其应用到 w13。
- 这使得 `w1`/`w3` 格式的 LoRA 文件可以被正确应用到打过补丁的模型（`w13`）中。

### 4. 保留 V2 以供测试

**策略**：

- 保留 V2 以配合自定义的非官方加载器进行测试。
- 提供 V3 作为新文件以支持 ComfyUI-nunchaku 官方的 Z-Image-Turbo 模型加载器。
- 两者均使用相同的 `compose_loras_v2`，但通过自动映射切换来正确运行。

**结果**：

- V3 适用于 ComfyUI-nunchaku 官方的 Z-Image-Turbo 模型加载器。
- 使用官方模型加载器时，用户应当使用 V3。

## 运行流程

### 使用 V3 节点时（ComfyUI-nunchaku 官方模型加载器）

- 节点执行：`NunchakuZImageTurboLoraStackV3.load_lora_stack()`
- 模型检测：检查 `model_wrapper_type_name == "NextDiT"`
- 包装器创建：使用 `ComfyZImageTurboWrapper` 包装 `NextDiT`
- LoRA 合成：调用 `compose_loras_v2()`
- 映射切换：使用 `_get_module_by_name()` 检测 `layers.0.feed_forward.w13`
- 映射应用：将 `_ACTIVE_KEY_MAPPING` 设置为 `ZIMAGE_NEXTDIT_NUNCHAKU_PATCHED_KEY_MAPPING`
- 键解析：在 `_classify_and_map_key()` 中使用 `_ACTIVE_KEY_MAPPING`
- LoRA 应用：应用 `w1`/`w3` → `w13`（GLU 融合），`to_q/k/v` → `qkv`（QKV 融合）

### 前向传播执行期间（NextDiT）

- 签名检测：使用 `inspect.signature()` 检查 `context` 和 `num_tokens`
- 分支处理：作为 NextDiT 格式处理
- 参数准备：将 `x` 统一为张量格式，并推导 `num_tokens`
- 调用：`model.forward(x, timestep, context=context, num_tokens=num_tokens, ...)`

## 总结

V3 的开发实现了以下目标：

- **支持 ComfyUI-nunchaku 官方模型加载器**：使 LoRA 能应用到通过 ComfyUI-nunchaku 官方 Z-Image-Turbo 模型加载器（`NextDiT` 格式）加载的模型上。
- **自动映射切换**：检测模型结构并自动选择适当的映射。
- **代码重用**：扩展了现有的 `compose_loras_v2` 和 `ComfyZImageTurboWrapper`，将代码重复降到最低。

**使用 ComfyUI-nunchaku 官方 Z-Image-Turbo 模型加载器时，请使用 V3 版本。**
