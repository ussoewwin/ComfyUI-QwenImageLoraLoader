## 概述

本文档完整说明 **ComfyUI-QwenImageLoraLoader** 中 `ComfyQwenImageWrapper` 的两项修复，对应 GitHub 发行 [v2.4.2](https://github.com/ussoewwin/ComfyUI-QwenImageLoraLoader/releases/tag/v2.4.2) 的英文说明：

1. **AttributeError**：与 Nunchaku Qwen Image 模型一起使用 Qwen Image Union ControlNet（例如 Fun ControlNet）时出现 `'ComfyQwenImageWrapper' object has no attribute 'process_img'`。
2. **RecursionError**：在 `NunchakuQwenImageLoraStackV3` 等代码中访问 `model_wrapper.model` 时出现 `maximum recursion depth exceeded`。

---

## 修改的文件

- **ComfyUI-QwenImageLoraLoader/wrappers/qwenimage.py**
- **类**：`ComfyQwenImageWrapper`（继承自 `nn.Module`）

---

## 问题 1：ControlNet 所需的 base_model 接口缺失

### 现象

- **错误**：`AttributeError: 'ComfyQwenImageWrapper' object has no attribute 'process_img'`
- **位置**：ComfyUI 的 `comfy/ldm/qwen_image/controlnet.py` 内 `QwenImageFunControlNetModel.forward`，在 `base_model.process_img(x)` 处
- **条件**：Nunchaku Qwen Image 模型经本加载器包装为 `ComfyQwenImageWrapper` 时，与 **Qwen Image Union ControlNet（Fun ControlNet）** 一起使用

### 调用路径（摘要）

```
KSampler
  → … → calc_cond_batch
    → controlnet.get_control
      → control_model(..., base_model=model, ...)
        → QwenImageFunControlNetModel.forward(..., base_model=...)
          → base_model.process_img(x)   ← AttributeError（当 base_model 为包装器时）
```

ComfyUI 的 `QwenImageTransformer2DModel`（及 Nunchaku 的 `NunchakuQwenImageTransformer2DModel`）具备 ControlNet 所需的全部接口；**包装器原先只委托了 `forward`，未暴露 `process_img` 及 `patch_size`、`pe_embedder`、`img_in`、`txt_norm`、`txt_in`、`time_text_embed` 等**。

### 根因摘要

- **原因**：`ComfyQwenImageWrapper` 只关注推理 `forward` 的委托，未提供 ControlNet 假定的 **base_model** 接口。
- **责任**：包装器侧必须满足与 Qwen Image 类模型相同的接口（本修复在包装器侧完成）。

---

## 修复 1

### 修复 1-A：新增 `process_img` 方法

**目标**：允许 ControlNet 调用 `base_model.process_img(x)`。

**新增代码**（`wrappers/qwenimage.py`）：

```python
def process_img(self, x, index=0, h_offset=0, w_offset=0):
    """
    委托给内部模型，以便 Qwen Image ControlNet（例如 Fun ControlNet）
    在 Nunchaku Qwen Image 模型被本包装器包装时，可以调用 base_model.process_img(x)。
    """
    if self.model is None:
        raise RuntimeError("Model has been unloaded. Cannot call process_img.")
    return self.model.process_img(x, index=index, h_offset=h_offset, w_offset=w_offset)
```

- 签名与返回值与 ComfyUI 的 `QwenImageTransformer2DModel.process_img` 一致（例如三元组 `(hidden_states, img_ids, orig_shape)`）。
- 此处通过**普通属性访问** `self.model`（不经过 `__getattr__`），避免问题 2 中的递归。

### 修复 1-B：通过 `__getattr__` 转发其他 base_model 属性

**目标**：当包装器上没有 `patch_size`、`pe_embedder`、`img_in`、`txt_norm`、`txt_in`、`time_text_embed` 等属性时，转发到内部模型。

**新增代码**（最终版，含问题 2 的递归修复）：

```python
def __getattr__(self, name):
    """
    将属性访问转发到内部模型，以便 Qwen Image ControlNet（例如 Fun ControlNet）
    在 base_model 为本包装器时，可以访问 patch_size、pe_embedder、img_in、
    txt_norm、txt_in、time_text_embed 等。
    通过 _modules 获取内部 模型，避免在 __getattr__ 内访问 self.model 导致递归。
    """
    try:
        inner = object.__getattribute__(self, "_modules").get("model")
    except (AttributeError, KeyError):
        inner = None
    if inner is None:
        raise AttributeError(f"'{type(self).__name__}' object has no attribute '{name}'")
    if name == "model":
        return inner
    return getattr(inner, name)
```

- ControlNet 在 `forward` 中使用的属性示例：`base_model.process_img(x)`、`base_model.patch_size`、`base_model.pe_embedder(ids)`、`base_model.img_in(...)`、`base_model.txt_norm(context)`、`base_model.txt_in(...)`、`base_model.time_text_embed(timesteps, ...)`。
- 当 `base_model` 为 `ComfyQwenImageWrapper` 时，上述调用可透明工作。

## 问题 2：`model_wrapper.model` 的 RecursionError

### 现象

- **错误**：`RecursionError: maximum recursion depth exceeded`
- **位置**：`nunchaku/lora/qwenimage_v3.py` 的 `load_lora_stack` 等处：`transformer = model_wrapper.model`
- **堆栈要点**（`wrappers/qwenimage.py` 第 85 行）：`if self.model is None:` 在 `__getattr__` 内再次触发 `__getattr__`

### 根因：`nn.Module` 与 `self.model` 的交互

- 子模块通过 `self.model = model` 注册在 **`_modules["model"]`**，不一定在 `__dict__["model"]`。
- 正常属性访问时，`Module.__getattribute__` 会查 `_modules`，`obj.model` 通常不会进入 `__getattr__`。
- **在 `__getattr__` 内写 `self.model` 时**，会再次为 `"model"` 调用 `__getattr__`，形成无限循环。

### 根因摘要

- **原因**：在 `__getattr__` 内引用 `self.model` 获取内部 transformer。
- **责任**：仅本包装器实现；应通过 `object.__getattribute__(self, "_modules").get("model")` 获取内部模型。

---

## 修复顺序与依赖关系

1. **问题 1**：`process_img` + `__getattr__` 属性转发（ControlNet 兼容）
2. **问题 2**：`__getattr__` 内用 `_modules` 取 `model`，避免 `self.model` 递归

两项修复**互不冲突**，可共存。

## 技术说明（ComfyUI ControlNet 参考）

`QwenImageFunControlNetModel.forward` 中对 `base_model` 的期望（节选）：

```python
hidden_states, img_ids, _ = base_model.process_img(x)
txt_start = round(
    max(
        ((x.shape[-1] + (base_model.patch_size // 2)) // base_model.patch_size) // 2,
        ((x.shape[-2] + (base_model.patch_size // 2)) // base_model.patch_size) // 2,
    )
)
image_rotary_emb = base_model.pe_embedder(ids).to(x.dtype).contiguous()
hidden_states = base_model.img_in(hidden_states)
encoder_hidden_states = base_model.txt_norm(context)
encoder_hidden_states = base_model.txt_in(encoder_hidden_states)
temb = (
    base_model.time_text_embed(timesteps, hidden_states)
    if guidance is None
    else base_model.time_text_embed(timesteps, guidance, hidden_states)
)
```

因包装器通过 `process_img` 和 `__getattr__` 转发上述接口，上述代码在 `base_model` 为 `ComfyQwenImageWrapper` 时**无需修改**即可工作。

## 总结

| 项目 | 问题 1（ControlNet） | 问题 2（RecursionError） |
|------|------------------------|---------------------------|
| 现象 | 缺少 `process_img` 等属性 | `model_wrapper.model` 无限递归 |
| 原因 | 包装器未提供 ControlNet 的 base_model 接口 | `__getattr__` 内使用 `self.model` |
| 修复 | 新增 `process_img` + `__getattr__` 转发 | `__getattr__` 通过 `_modules["model"]` 取内部模型 |
| 文件 | `wrappers/qwenimage.py` | 同上 |

完成上述两项修复后：

- **Qwen Image Union ControlNet（例如 Fun ControlNet）** 可与 Nunchaku Qwen Image 模型一起使用。
- 在 `NunchakuQwenImageLoraStackV3` 等中通过 **`model_wrapper.model`** 获取内部 transformer 时不再递归。
